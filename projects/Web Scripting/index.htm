Welcome to Web Scripting

The aim of this Module is to provide the student with the necessary skills to enable them to develop an Internet site that utilises client- side JavaScript techniques and simple server-side techniques.

Learning Outcomes

By the end of this module the student should be able to :

  1. Demonstrate an understanding of client side functionality that can provide well-structured and usable applications.
  2. Compare the use of CSS and Javascript within web pages.
  3. Demonstrate an understanding of the underlying Document Object Model within web pages, and implement foundational methods of manipulating content via the DOM.
  4. Demonstrate an understanding of simple server side scripting, and produce web pages that connect an application to a database, work with data and display data.

Indicative Content

Overview of Client Side Scripting

JavaScript's bad reputation. DHTML preceived as bad practice, DOM perceived as good practice. Major scripting standards. The Browser Object Model and the Document Object Model. Syntax and features. Browser (in)compatibilities, security issues and common speed and load issues. Running and testing scripts.

The DOM

Abstraction. Reaching Nodes - getelementById and getElementByTagName. Attributes of nodes - data, nodeName, nodeType, nodeValue, getAttribute (attribute). Working with nodes - document.createTextNode etc, child nodes, parent nodes, sibling nodes, etc.

JavaScript Procedural Programming

Script tags. Statements, expressions and operators, line termination and the semicolon. Variables. Comments in JavaScript.

JavaScript Functional Programming

User-Defined Functions. Built in Functions (Maths functions, Timers, message boxes, confirmations, String manipulation).

JavaScript Event-Driven Programming

Detecting and invoking actions on the detection of events.

JavaScript Objects

Built In Objects, Date use, document / DOM, window. Functions as objects.

JavaScript Advanced Techniques

Cookies for storing and retrieving data. Pre-fetching content. Exception handling.

Graceful Degradation

Designing JavaScript so that it is not essential to the key functionality of a site. Shifting event handlers and other JavaScript out of the html file and into the js file. Alternatives in the event of there being no JavaScript support.

The client/server Model

Comparing static web pages to dynamic web pages. the rolw of the web server and application server. Understanding the sequence of events and limitations of the standard model.

Basic Server-Side Concepts

Providing dynamic content using a database and an application server. Connecting web pages and databases and using simple and parameter queries to extracting data for display (SELECT queries) and maintain data (INSERT, UPDATE, DELETE queries). Elementary display mechanisms.
